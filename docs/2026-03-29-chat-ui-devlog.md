## 本次最终保留的 UI 调整

### 1. 左侧助手/话题区域结构收紧
- 继续收窄左侧栏顶部净空，让 `助手 / 话题 / 设置` tab、搜索栏、列表内容整体更靠上
- 删除了助手页中的 `VCP Agents` 伪列表头，让搜索和列表直接顶上来
- 删除了话题页原有的“话题”标题行，让话题页结构与助手页保持一致

### 2. 左侧搜索栏与列表项风格统一
- 将助手页搜索栏和话题页搜索栏统一为与列表项同节奏的胶囊结构
- 搜索栏加入左侧搜索图标占位，并对齐列表项头像起始位
- 搜索栏高度、横向外边距、与下方列表的垂直间距都按列表节奏重新调整
- 常态下保留外框显示，悬停和聚焦时继续增强玻璃质感
- 在浅色和深色模式下都为搜索栏轻微混入主题色，但强度明显弱于列表选中态
- 额外收掉了搜索胶囊底边过亮的白色高光，避免出现底部一丝明显白边

### 3. 助手列表与话题列表的胶囊化统一
- 助手列表项继续维持更紧凑的胶囊尺寸和更统一的横向留白
- 话题列表样式整体向助手列表靠拢，包括搜索、条目密度和文本排版
- 去掉了话题选中态最左侧的彩色竖条特效，减少干扰
- 话题数量小胶囊改成更明显的液态玻璃样式，并将尺寸再缩小一档

### 4. 左下角创建按钮重做
- `创建Agent` 和 `创建Group` 两个按钮改为图标 + 文本结构，而不是纯文本按钮
- 为两个按钮分别接入新的内联 SVG 图标
- 按钮常态改为液态玻璃风格，并与右上角按钮体系保持一致的视觉语言
- 深色模式下按钮常态文字/图标颜色与“与 xxx 聊天中”使用同一主题色
- 悬停态改为更实的强调色填充，同时移除了“往上提”的 hover 位移感
- 在侧栏缩窄时保留图标与 `Agent / Group` 关键词，确保窄宽度下仍能识别
- 对 Group 文本做了轻微上移修正，缓和与 Agent 的基线差异

### 5. 右上角功能按钮细调
- 顶部按钮保留当前圆角矩形方案，只加大圆角，不改成纯圆/药丸
- 浅色模式下选中态边框改为主题色，不再使用之前偏红的硬编码色
- 深色模式下默认文字/图标颜色改为与“与 xxx 聊天中”同色
- 深色模式下 hover 状态与左下角创建按钮的 hover 逻辑对齐
- 第四个按钮保留定制 `CREATE / TOPIC` SVG，并将文字字重继续下调到更轻的版本

### 6. 聊天头部与主聊天区微调
- 保留当前实色/毛玻璃 `chat-header` 方案，不继续使用渐变融入式 header
- 在此前基础上维持更收窄的 header 高度，减少顶部占用
- 缩小“选择一个 Agent 或群组开始聊天”以及“与 xx 聊天中”标题的视觉存在感

### 7. 输入区外框与内部布局微调
- 将大输入框外层 `.chat-input-card` 增加轻微玻璃质感，但保持克制
- 调整聊天输入区域整体底部留白，使其与左侧栏底部按钮更接近对齐
- 调整输入区内部的内容分布：
  - 增加外层卡片顶部内边距
  - 缩小输入区与工具按钮行之间的间距
  - 轻微下移输入内容与工具按钮整体的视觉重心

### 8. 侧栏与辅助交互逻辑同步
- `main.html`、`modules/event-listeners.js`、`modules/ui-helpers.js` 同步更新为新的创建按钮结构
- 保证在动态重绘/设置面板切换后，创建按钮仍能恢复为新的图标 + 文本结构

### 9. 主题变量继续演进
- `styles/themes.css` 当前工作树保留了一组新的主题变量和壁纸资源定义
- 这些调整与本次 UI 微调一起进入本次提交，作为主题层持续演进的一部分

## 本次尝试后已回退的方案
- 聊天区顶部导航改为“渐变融入内容”的方案已再次回退，当前仍保留实色/毛玻璃 header
- 聊天记录顶部做淡出遮罩的方案已取消
- 右上角五个按钮整组强化为更重液态玻璃风格的方案已回退
- 右上角按钮改为纯圆/药丸组合的方案已回退
- 为排查顶部磨砂断层临时禁用 `seam-fixer` 的方案已回退
- 对左下角 `Agent / Group` 文本与图标做过多轮 1px 级对齐实验，其中偏差过大的实验版本已回退
